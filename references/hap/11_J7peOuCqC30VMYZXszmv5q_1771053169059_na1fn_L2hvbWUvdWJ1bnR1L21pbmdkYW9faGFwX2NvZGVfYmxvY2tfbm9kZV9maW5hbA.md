# 明道云HAP代码块节点模块深度解析

## 1. 功能概述

明道云HAP（Hyper Application Platform）中的**代码块节点**是工作流自动化中的一个核心组件，它允许用户在无代码/低代码环境中嵌入自定义的JavaScript或Python代码，对流程中的数据进行个性化处理、复杂计算或高级逻辑判断。通过代码块节点，用户可以突破平台预设功能的限制，实现高度定制化的业务需求，并将处理后的新数据内容输出，供后续节点使用。它赋予了工作流“编程大脑”，使得自动化流程更加智能和灵活 [1]。

**业务价值：**
*   **数据个性化处理：** 针对特定业务场景，对数据进行清洗、格式转换、复杂计算等。
*   **动态逻辑判断：** 实现基于多条件、复杂规则的流程分支或数据筛选。
*   **高级集成能力：** 调用外部API（支持WebService SOAP协议，参数格式支持XML/JSON/Form-Data，请求方式支持GET/POST/PUT/DELETE），实现与第三方系统的深度集成。
*   **弥补平台功能不足：** 在明道云HAP现有节点无法满足特定需求时，通过代码块进行功能扩展。

## 2. 使用场景

代码块节点广泛应用于需要自定义数据处理和逻辑判断的各种业务场景，以下是一些典型应用案例：

*   **数据清洗与格式化：**
    *   **场景1：** 身份证信息提取：根据用户输入的身份证号码，自动解析出性别、出生日期、年龄等信息，并填充到工作表对应字段 [1]。
    *   **场景2：** 地址信息标准化：将用户输入的非标准地址格式化为统一的省、市、区、街道等字段。
    *   **场景3：** 文本内容处理：对用户输入的文本进行关键词提取、敏感词过滤或内容摘要。

*   **复杂业务逻辑计算：**
    *   **场景4：** 订单金额计算：根据商品数量、单价、折扣、运费等多个变量，计算最终订单总价，并考虑不同会员等级的优惠策略。
    *   **场景5：** 绩效考核评分：根据员工的多个考核指标（如销售额、客户满意度、项目完成率），通过自定义权重和算法计算最终绩效得分。
    *   **场景6：** 库存预警与调拨：根据当前库存量、销售预测和安全库存阈值，计算是否需要进行库存补充或跨仓库调拨。

*   **外部系统集成与API调用：**
    *   **场景7：** 调用第三方物流API：在订单发货后，通过代码块调用物流公司的API，自动获取运单号和物流状态，并更新到明道云工作表。
    *   **场景8：** 财务系统对接：将明道云中的审批通过的报销单数据，通过代码块调用企业财务系统的API，自动创建财务凭证。
    *   **场景9：** 短信/邮件服务集成：使用代码块调用第三方短信或邮件API，发送个性化通知，例如验证码、营销信息等（尽管明道云有内置节点，但代码块可提供更灵活的定制）。

*   **动态流程控制：**
    *   **场景10：** 动态审批人指定：根据表单中的特定字段值（如金额大小、部门、项目类型），通过代码块动态判断并指定下一环节的审批人或审批组。
    *   **场景11：** 复杂条件分支：实现比工作流内置分支节点更复杂的条件判断，例如结合多个字段、外部数据源进行决策。

## 3. 操作指南

本节将详细介绍如何在明道云HAP工作流中添加、配置和使用代码块节点。

### 3.1 添加代码块节点

1.  进入明道云HAP应用，点击左侧导航栏的“工作流”进入工作流设计界面。
2.  选择或创建一个工作流，进入工作流编辑画布。
3.  在左侧节点库中，找到“系统工具”分类下的“代码块”节点，将其拖拽到工作流画布中。

### 3.2 配置代码块节点

双击画布上的代码块节点，打开配置面板。主要配置项包括：

#### 3.2.1 选择代码语言

在代码块配置面板顶部，选择您希望使用的编程语言：**JavaScript** 或 **Python**。平台会根据选择提供对应的运行环境和预装库。

#### 3.2.2 定义输入参数

代码块节点可以接收来自工作流上游节点的数据作为输入。您需要定义这些输入参数的名称和值。

1.  在“输入参数”区域，点击“+ Key/Value Pair”添加新的参数。
2.  在“Key”字段中输入参数的名称（建议使用英文命名，避免中文可能导致的兼容性问题）。
3.  在“Value”字段中，您可以选择：
    *   **固定值：** 直接输入一个常量值。
    *   **动态值：** 点击输入框右侧的“+”号，从工作流上游节点中选择需要传入的字段值。例如，选择“获取单条数据”节点输出的“身份证号”字段。

**在代码块中接收输入参数的格式：**
*   **JavaScript：** `input.参数名` (例如：`input.IDCard`)
*   **Python：** `input[
参数名"]` (例如：`input["IDcard"]`)

#### 3.2.3 编写代码

在代码编辑器中，根据您的业务需求编写JavaScript或Python代码。您可以利用传入的参数进行计算、处理和逻辑判断。

#### 3.2.4 定义输出参数

代码处理完成后，需要将结果输出，供工作流后续节点使用。

**输出结果的格式：**
*   **JavaScript：** `output = {参数名1: 参数值1, 参数名2: 参数值2};`
*   **Python：** `output = {'参数名1': 参数值1, '参数名2': 参数值2}`

**注意：** 输出参数的名称（Key）同样建议使用英文，以保证最佳兼容性。

### 3.3 测试代码块

1.  代码编写完成后，点击配置面板下方的“测试”按钮。
2.  如果您的输入参数是动态值，系统会提示您输入一个测试值。
3.  点击“执行”，系统将运行您的代码并显示输出结果。
4.  如果代码执行成功并返回了预期的输出，说明代码逻辑正确。如果报错，请根据错误提示检查您的代码。

> **重要提示：** 只有代码块节点成功测试并保存后，其输出参数才能在后续节点中被引用。

### 3.4 在后续节点中使用输出

代码块节点成功执行后，其输出的参数就可以在工作流的后续节点中作为动态值来使用。例如，在一个“更新记录”节点中，您可以将代码块输出的“生日”和“性别”分别更新到工作表的对应字段中。

## 4. 字段/选项说明

下表详细说明了代码块节点配置界面中的所有可配置项及其使用建议。

| 配置项 | 说明 | 使用建议 |
| --- | --- | --- |
| **节点名称** | 在工作流画布上显示的节点名称。 | 使用能够清晰表达该代码块功能的名称，例如“计算订单总价”、“解析身份证信息”。 |
| **代码语言** | 选择使用JavaScript或Python。 | 根据您团队的技术栈和代码片段库的积累来选择。Python在数据处理方面有更强大的库支持。 |
| **输入参数 (Key-Value)** | 定义传入代码块的数据。Key是参数名，Value是参数值。 | Key使用简洁明了的英文名称。Value优先使用上游节点的动态值，以实现流程自动化。 |
| **代码编辑器** | 编写和编辑代码的核心区域。 | 利用编辑器内置的语法高亮功能。对于复杂逻辑，建议先在本地IDE中编写和测试。 |
| **输出参数 (output)** | 定义代码块执行后返回给工作流的数据。 | 输出的Key同样使用英文。确保输出的数据结构与后续节点的需求一致。 |
| **测试按钮** | 用于在保存前验证代码的正确性。 | 每次修改代码后都必须进行测试，确保逻辑无误。 |
| **自动重试** | 当代码块执行失败时，是否自动重试一次。 | 对于涉及外部API调用等可能存在网络波动的场景，建议勾选此项以增加流程的健壮性。 |
| **代码片段库** | 保存和复用常用的代码片段。 | 将通用性高、复用性强的代码保存到组织片段库，提高团队开发效率。 |
| **AI生成代码** | 根据自然语言描述自动生成代码。 | 用于快速生成简单的、常见的代码逻辑，可作为开发的起点，但生成后仍需仔细审查和测试。 |

## 5. 开发规范

为了保证代码的可读性、可维护性和执行效率，建议遵循以下开发规范：

*   **命名规范：**
    *   输入和输出参数的Key，以及代码内部的变量名，全部使用**驼峰式命名法**（camelCase），例如 `orderAmount`, `customerName`。
    *   命名应清晰、简洁，能够准确描述其代表的数据或功能。
*   **注释规范：**
    *   对于复杂的逻辑、算法或外部API调用，必须添加清晰的注释，解释其功能和实现方式。
    *   在代码开头，建议添加注释说明该代码块的整体功能、输入和输出。
*   **错误处理：**
    *   在代码中应充分考虑异常情况，例如输入数据为空、格式错误或外部API调用失败。
    *   使用 `try...catch` (JavaScript) 或 `try...except` (Python) 结构来捕获和处理潜在的错误，避免整个工作流因意外中断。
*   **性能考虑：**
    *   避免在代码块中进行大规模的数据查询和循环操作。如果需要处理大量数据（例如超过100条），应考虑使用工作流的“循环”节点或“获取多条数据”节点的批量处理能力。
    *   代码块从“获取多条数据”节点接收的数据上限为100行，设计时需注意此限制。

## 6. 最佳实践

*   **保持原子性：** 一个代码块节点应只专注于完成一个独立的、单一的功能。避免将多个不相关的逻辑混在一个代码块中，这有利于复用和维护。
*   **善用代码片段库：** 将常用的、经过验证的代码逻辑（如日期格式化、加解密算法、特定API调用等）保存到代码片段库，实现“一次编写，多次使用”。
*   **充分利用预装库：** 明道云HAP为JS和Python环境预装了大量常用库（见下表），应优先使用这些库来解决问题，而不是自己“造轮子”。

| 语言 | 预装库 |
| --- | --- |
| **Node.js v10.16.3** | `moment`, `lodash`, `node-fetch`, `form-data`, `xml2js`, `node-rsa`, `uuid`, `pinyin`, `sm-crypto` |
| **Python v3.7.5** | `requests`, `xmltodict`, `pycryptodome`, `dingtalk-sdk`, `beautifulsoup4`, `rsa`, `uuid`, `xpinyin`, `gmssl`, `cryptography`, `lxml`, `pandas`, `numpy`, `matplotlib` |

*   **日志与调试：** 在复杂的代码块中，可以使用 `console.log()` (JavaScript) 或 `print()` (Python) 输出中间变量的值。这些输出会显示在代码块的测试结果中，便于调试。
*   **安全第一：** 在处理来自用户输入的数据时，要警惕潜在的安全风险，例如代码注入。虽然明道云平台有一定的沙箱机制，但仍需对输入数据进行校验和清理。

## 7. 与其他模块的配合

代码块节点可以与工作流中的几乎所有其他节点协同工作，形成强大的自动化能力。

*   **与“获取数据”节点的配合：** “获取单条/多条数据”节点为代码块提供输入数据源，代码块处理后，再由“更新记录”节点将结果写回工作表。
*   **与“分支”节点的配合：** 代码块可以实现复杂的逻辑判断，并将判断结果（例如 `true` 或 `false`）输出，供后续的“分支”节点使用，从而决定流程走向。
*   **与“调用已集成API”节点的配合：** 虽然两者都可以调用API，但代码块提供了更高的灵活性。您可以在调用API前后进行数据处理，或者实现更复杂的认证逻辑。如果API已在集成中心配置好，优先使用“调用已集成API”节点；反之，或需要更灵活的控制时，使用代码块。
*   **与“循环”节点的配合：** 当需要对多条记录逐一进行复杂处理时，可以使用“循环”节点，在循环内部嵌套一个代码块节点，实现对每条记录的个性化操作。

## 8. 实际案例：动态计算订单折扣

**业务场景：** 根据客户的会员等级和订单金额，动态计算折扣，并更新订单最终价格。

*   **会员等级与折扣策略：**
    *   普通会员：订单满100元，9.8折
    *   白银会员：订单满200元，9.5折
    *   黄金会员：订单满300元，9折

**实现步骤：**

1.  **触发流程：** 工作流由“当新增订单记录时”触发。
2.  **获取客户信息：** 使用“获取单条数据”节点，根据订单中的客户ID，从“客户信息表”中获取该客户的“会员等级”。
3.  **配置代码块节点：**
    *   **节点名称：** “计算订单折扣”
    *   **输入参数：**
        *   `orderAmount`: 订单金额（来自触发节点）
        *   `membershipLevel`: 会员等级（来自“获取客户信息”节点）
    *   **代码 (JavaScript)：**

        ```javascript
        let discount = 1.0;
        const amount = input.orderAmount;
        const level = input.membershipLevel;

        if (level === '普通会员' && amount >= 100) {
            discount = 0.98;
        } else if (level === '白银会员' && amount >= 200) {
            discount = 0.95;
        } else if (level === '黄金会员' && amount >= 300) {
            discount = 0.9;
        }

        const finalAmount = amount * discount;

        output = {
            finalAmount: finalAmount,
            discountApplied: 1 - discount
        };
        ```

4.  **更新订单记录：** 添加一个“更新记录”节点，将代码块输出的 `finalAmount`（最终金额）和 `discountApplied`（折扣率）更新到当前订单记录的对应字段中。

通过这个案例，我们看到代码块节点如何将复杂的、多条件的业务规则，以一种灵活且可维护的方式固化到自动化流程中，这是简单的无代码配置难以实现的。
