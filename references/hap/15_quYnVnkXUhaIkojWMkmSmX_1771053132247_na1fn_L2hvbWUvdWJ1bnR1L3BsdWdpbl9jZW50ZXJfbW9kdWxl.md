# 明道云HAP插件中心模块深度解析


## 1. 功能概述

### 什么是视图插件？
视图插件又叫「自定义视图」，当HAP的表格、看板、层级、日历、画廊、详情、甘特图等系统视图不能满足用户视图展示需求的时候，开发者可以通过自己编写代码实现一个完全自定义的视图页面，用于展示工作表的记录数据。自定义视图支持搜索、筛选、统计、快速筛选和筛选列表等操作，还可以通过HAP公共Javascript接口实现调用系统组件，比如展示记录详情弹窗、调用新建记录窗口等等。

## 2. 使用场景

### 视图插件和系统视图有什么区别？
从使用者的角度看，视图插件和普通视图是没有任何区别的。当组织管理员通过发布开发者插件、安装插件或者导入插件后，所有已启用的插件即对组织下的所有用户生效，用户可以像使用表格、看板、日历等系统视图一样使用这些视图，也可以正常的为视图分配权限和进行视图分享等操作。

### 插件需求分析
在制作视图插件之前，一定要对要开发的视图进行需求分析，明确视图的适用范围，并通过设计合理的设置项来提高视图插件的通用性。

**示例场景：**
有两张工作表：订单和订单明细。开发者想自己开发一个视图，将「订单明细」的数据显示到主「订单」的表格中，主订单的数据将以合并单元格的方式同时展示两张表的数据。

## 3. 操作指南

### 开发步骤

#### 准备工作
*   安装 Node.js（>=16.20） 和 npm
*   准备集成开发环境（IDE），推荐 VS Code
*   如果是团队开发，请准备好代码版本管理工具，推荐 Git

#### 创建视图插件
要创建一个视图插件，有两种方式：

1.  **通过新建视图时创建「自定义视图」**：系统会自动创建一个视图插件，并以当前工作表为该视图的开发调试环境。
2.  **在系统首页进入「插件中心」**：在插件「我开发的」页面中点击「制作插件」。通过此方式创建插件时，仍然需要选择一张工作表作为开发调试环境，选择后会自动在该工作表下创建一个新的视图用于开发调试视图插件。

创建好插件后，进入到工作表下新创建的这个自定义视图，可以进行下一步开发。

#### 插件基础设置
*   **插件名称**：建议能准确表达视图的作用，且不用带“视图”二字，比如：「地图」、「思维导图」、「树型表格」等等。
*   **图标**：可以使用自定义图标，相当于插件的logo。
*   **快速筛选和筛选列表**：视图插件允许用户自由选择是否启用。当启用后，视图使用者在操作了快速筛选项和筛选列表后，系统将向插件发送事件触发消息，开发者需要在插件中添加事件处理句柄并通过传入的筛选条件处理数据筛选逻辑。可以参考本文档附录中的 [mdye 消息系统](mdye 消息系统) 示例代码。

#### 本地项目创建与调试
1.  **选择脚手架模板**：系统提供了React 基础示例模板、JavaScript 基础示例模板、React + Tailwind CSS 模板、Vue 3 模板、Vue 2 模板。
2.  **安装HAP命令行工具 `mdye`**：
    ```bash
    $ npm install -g mdye-cli
    # 如果报没有权限的错误，请用 `sudo` 来安装：
    $ sudo npm install -g mdye-cli
    ```
    验证安装：`$ mdye --version`
3.  **创建本地项目**：在「建立本地项目」步骤中复制创建插件本项目的命令，在本地终端中执行。
4.  **启动本地项目调试**：
    ```bash
    $ cd your_project_folder # 进入项目文件夹
    $ code . # 在 VS code 中打开项目
    $ npm i # 安装项目依赖
    $ mdye start # 启动本地项目调试
    ```
    项目运行成功后，会生成一个本地服务器js文件，把这个地址填入插件的「调试」页面，并点击「加载」。

## 4. 字段/选项说明

### 设置项参数类型
| 设置项类型 | 子类型 | 值类型 | 备注 |
| --- | --- | --- | --- |
| 字段选择器 | 字段单选 字段多选 | array\[string\] | 字段多选时，可以限制选择字段的数量 |
| 字符串 | string |
| 数值 | double |
| 枚举值 | 单选框 下拉菜单 | array\[string\] | 选项格式为 key=value ，其中 vlaue 为呈现给使用者的文字，key 为代码中获取到的值； 样式为单选框时，可以选择横向或竖向排列； |
| 布尔值 | 开关 勾选框 | boolean |
| 分组标题 | null |

### 参数映射配置示例
在插件需求分析的示例中，定义了两个视图设置项：「显示字段(`showFields`)」和「子表明细字段(`sub`)」。在「参数映射」中，开发者可以将实际的视图配置映射到字段里，并在代码中通过 `env` 变量获取到它的值。

```javascript
import { env } from "mdye";
const { showFields, sub } = env;
// showFields, sub 即为使用者配置的值，变量名称和配置中的变量ID一一对应
```

## 5. 开发规范

*   **插件名称**：建议能准确表达视图的作用，且不用带“视图”二字，比如：「地图」、「思维导图」、「树型表格」等等。
*   **代码版本管理**：如果是团队开发，推荐使用 Git。
*   **前端开发基础**：需要开发者具备前端开发基础，掌握Javascript、CSS和HTML等相关知识。
*   **React JS**：如果开发者熟悉 React JS，也可以通过查看HAP前端开源项目 [https://github.com/mingdaocom/pd-openweb]() ，参考HAP系统视图代码进行插件的开发。

## 6. 最佳实践

*   **需求分析先行**：在制作视图插件之前，一定要对要开发的视图进行需求分析，明确视图的适用范围，并通过设计合理的设置项来提高视图插件的通用性。
*   **通用性设计**：通过对视图需求的整理，可以让用户更加明确开发的目标和实现的边界，也更容易将插件做到适应更多的通用场景，降低开发的成本。
*   **利用系统组件**：通过HAP公共Javascript接口实现调用系统组件，比如展示记录详情弹窗、调用新建记录窗口等等。

## 7. 与其他模块的配合

*   **HAP公共Javascript接口**：视图插件可以通过HAP公共Javascript接口实现调用系统组件，比如展示记录详情弹窗、调用新建记录窗口等等。
*   **mdye 消息系统**：当启用快速筛选和筛选列表后，系统将向插件发送事件触发消息，开发者需要在插件中添加事件处理句柄并通过传入的筛选条件处理数据筛选逻辑。
*   **工作表**：视图插件用于展示工作表的记录数据。

## 8. 实际案例

**案例：订单与订单明细合并显示**

**业务场景：** 开发者希望在一个视图中，将主「订单」工作表的数据与关联的「订单明细」工作表数据合并显示，其中主订单数据以合并单元格的方式展示。

**实现思路：**
1.  **功能实现**：采取先加载主表格，再通过异步的方式获取子表数据进行加载。
2.  **通用性配置**：
    *   允许使用者自行调整配置视图表格的显示字段和顺序。
    *   允许使用者配置要将哪张子表展示到主表格中。

## 1. 功能概述

插件中心是面向开发者对HAP系统功能进行扩展的能力。在插件中心，用户可以管理自己开发的插件，还可以查看并使用组织里其他开发者开发的所有插件。目前支持自定义开发视图和工作流的节点，并将陆续上线工作表字段插件、自定义页面插件。

## 2. 使用场景

*   **管理和使用插件**：用户可以在插件中心管理自己开发的插件，并使用组织内其他开发者开发的插件。
*   **扩展HAP功能**：通过开发视图插件、工作流节点插件、工作表字段插件和自定义页面插件，扩展HAP系统的功能，满足特定的业务需求。
