# 明道云HAP飞书集成模块深度研究

## 1. 功能概述

明道云HAP的飞书集成模块允许企业将HAP平台与飞书（Lark）进行深度整合。通过自建应用的方式，HAP能够实现以下核心功能：

*   **组织架构同步**：将飞书中的组织架构直接同步到HAP平台，确保两边通讯录信息的一致性。
*   **应用消息推送**：将HAP内部的应用消息和工作流消息实时推送到飞书客户端，方便用户及时接收和处理。
*   **飞书工作台集成**：将HAP中的单个应用安装到飞书工作台，用户可以直接从飞书点击图标进入HAP应用或指定工作表页面，实现统一入口。
*   **流程待办同步**：将HAP中的流程待办（如待审批、待填写）同步至飞书审批中心，并在飞书端处理后同步更新状态。
*   **飞书扫码登录**：支持用户通过飞书APP扫码或已登录账号验证登录HAP，提升登录便捷性。

该模块旨在解决企业内部系统割裂、信息孤岛、多平台操作繁琐等业务问题，通过集成实现数据互通、流程协同和用户体验优化，从而提升整体运营效率和员工工作效率。

## 2. 使用场景

明道云HAP飞书集成模块适用于多种业务场景，以下为具体案例：

1.  **企业通讯录统一管理**：当企业同时使用飞书作为日常沟通工具和HAP作为业务管理平台时，可以通过飞书集成模块，将飞书的组织架构同步至HAP，避免手动维护两套通讯录，确保员工信息的准确性和一致性。例如，新员工入职后，只需在飞书添加，HAP便自动创建账户。
2.  **跨平台消息通知**：HAP中的业务流程（如请假审批、报销流程、任务指派等）产生待办或通知时，可以实时推送到飞书，员工无需切换应用即可在飞书接收并处理，提高响应速度。例如，销售人员在HAP中提交的订单审批，审批人可在飞书收到通知并直接审批。
3.  **统一工作入口**：企业希望员工在一个平台（飞书）内完成大部分工作，HAP应用可以作为飞书工作台的一部分，员工点击飞书工作台中的HAP应用图标即可直接进入HAP的特定模块或页面，无需记住多个系统入口。例如，HR部门将HAP中的考勤应用集成到飞书工作台，员工每天从飞书直接打卡。
4.  **飞书审批流程协同**：对于HAP中涉及审批的业务流程，可以将其待办同步到飞书审批中心。员工可以在飞书审批中心统一查看和处理所有审批任务，审批状态实时同步回HAP，实现审批流程的无缝衔接。例如，采购申请在HAP中发起，审批待办同步到飞书，领导在飞书完成审批。
5.  **便捷的身份认证**：企业员工习惯使用飞书进行身份认证，通过飞书扫码登录HAP，可以省去输入账号密码的步骤，提高登录效率和安全性。尤其适用于移动办公场景，员工通过手机飞书APP快速登录HAP。
6.  **精细化通讯录同步控制**：企业可能存在部分人员（如实习生、兼职人员）不希望同步到HAP的情况。通过在飞书组织架构中添加自定义字段“同步”并设置为“不同步”，可以实现对特定人员的排除，保证HAP通讯录的精准性。

## 3. 操作指南

HAP与飞书的集成主要分为以下几个步骤：

### 3.1 准备工作

1.  **获取HAP对接信息**：登录HAP管理后台，点击【个人头像】 > 【组织管理】 > 【集成】 > 【企业身份】，选择“飞书”进入集成页面。在此页面，点击【如何获取对接信息？】以获取Logo、主页地址等对接所需信息。请勿关闭此页面，后续配置会用到。
2.  **登录飞书开放平台**：访问飞书开发者后台 [https://open.feishu.cn/app]() 或Lark开发者后台 [https://open.larksuite.com/app]()，并点击“创建企业自建应用”。

### 3.2 创建飞书应用

1.  **填写应用基本信息**：根据HAP对接信息页面提供的Logo、应用名称等，填写飞书应用的基本信息。
2.  **进入应用详情页**：创建应用后，打开刚才创建的飞书应用详情页。

### 3.3 配置飞书应用

1.  **设置主页地址**：
    *   在飞书应用中，添加应用能力，选择添加“网页应用”。
    *   在“网页应用”中，将HAP对接信息页面提供的主页地址分别添加到“桌面端主页”和“移动端主页”。
    *   在“安全配置”中，将HAP对接信息页面提供的主页地址添加到“重定向URL”。
2.  **启用机器人**：通过“应用能力” - “添加应用能力”，添加“机器人”。启用机器人后，HAP内部的待办通知、应用消息将能直接推送到飞书消息侧。
3.  **添加API权限**：
    *   在“开发配置”-“权限管理”中，点击“开通权限”按钮，添加**15**个接口的权限（应用身份权限）。
    *   逐个搜索并勾选以下接口权限：
        *   获取应用信息
        *   获取通讯录基本信息
        *   获取通讯录部门组织架构信息
        *   获取部门基础信息
        *   获取部门组织架构信息
        *   获取用户组织架构信息
        *   获取用户基本信息
        *   获取用户邮箱信息
        *   获取用户受雇信息
        *   获取用户 user ID
        *   获取用户手机号
        *   以应用的身份发消息
        *   给多个用户批量发消息
        *   查看、创建、更新、删除审批应用相关信息
        *   查看、创建、更新、删除三方审批定义相关信息
    *   **注意**：务必选择**应用身份权限**类别下的接口权限，避免误选用户身份权限。
4.  **配置发布版本和可用人员范围**：
    *   **创建发布版本**：应用配置完毕后，需要创建发布版本。如果是更新版本，版本号需递增（如1.0.0改为1.0.1）。
    *   **配置同步人员范围和应用可见人员范围**：这里选择的人员和部门，决定了在飞书中可见“HAP”应用的人员，也是同步到HAP的人员和部门。例如，如果公司有100名飞书用户，但只有30名HAP用户，则在此处选择这30名用户或其所在部门。
    *   **申请发布**：每次调整应用后，都需要创建新版本，发布并提交给飞书管理员审核。确保版本状态为**已发布**才能正常使用。建议请管理员将此应用设置为免审核，以加快发布流程。

### 3.4 完善HAP对接信息

1.  **获取App ID和App Secret**：从飞书应用详情页获取App ID和App Secret，并将其填写到HAP集成页面的相应位置。

### 3.5 同步通讯录

1.  **同步规则**：飞书通讯录同步到HAP时，主要依据手机号和邮箱（优先手机号）进行匹配关联。如果HAP组织中存在相同手机号的账户，则直接建立关联；否则，自动创建新的HAP账户。若手机号匹配不上，会尝试按邮箱匹配。
2.  **同步前注意事项**：
    *   确定使用HAP的飞书账号范围（参考3.3.4步）。
    *   已加入HAP组织的员工，需核对HAP账户和飞书账户绑定的手机号是否一致，不一致请及时修改。
    *   尚未拥有HAP账户的员工，同步后会自动创建HAP账户并绑定随机生成的邮箱。
3.  **执行同步**：组织超级管理员进入HAP管理后台 > 集成 > 第三方平台，点击【同步】按钮。
4.  **指定人员不同步**：若需排除部分人员不同步到HAP，可在飞书/Lark组织架构中添加自定义文本信息字段，命名为“同步”，并将其设置为“不同步”。
    *   **添加自定义字段**：在飞书/Lark管理后台 > 组织架构 > 字段管理 > 自定义字段，新增字段“同步”，归属于“成员”，类型为“文本”。
    *   **开启API调用**：开启“允许开放平台通讯录API调用”。
    *   **设置不同步成员**：将不希望同步的成员的“同步”字段设置为“不同步”。

### 3.6 其他集成功能配置

1.  **消息打开方式**：在HAP管理后台 > 企业身份 > 其他，可选择飞书接收HAP消息的打开方式：独立窗口打开（通过浏览器）或侧边栏打开（在飞书内）。
2.  **流程待办同步至飞书审批中心**：
    *   **添加审批权限**：在飞书对接应用中添加“查看、创建、更新、删除审批应用相关信息”和“查看、创建、更新、删除三方审批定义相关信息”两个权限，并重新发布应用。
    *   **飞书APP添加审批应用**：在飞书APP中添加审批应用。
    *   **HAP中开启同步**：在HAP中开启流程待办同步至飞书审批中心。
3.  **飞书扫码登录**：
    *   **配置二级域名**：在HAP组织管理后台配置二级域名（例如`mega.mingdao.com`）。
    *   **启用扫码登录**：启用“飞书扫码登录”功能。
    *   **添加重定向URL**：在飞书对接应用 > 开发配置 > 安全设置 > 重定向URL中，添加回调地址（例如`https://mega.mingdao.com/social/feishu/callback`）。

## 4. 字段/选项说明

| 字段/选项名称 | 说明 | 使用建议 |
| :---------- | :--- | :------- |
| **App ID** | 飞书应用唯一标识 | 从飞书开放平台获取，用于HAP识别飞书应用。 |
| **App Secret** | 飞书应用密钥 | 从飞书开放平台获取，用于HAP与飞书进行安全通信。 |
| **主页地址** | HAP应用在飞书中的访问地址 | 确保桌面端和移动端主页地址均正确配置，保证多端访问。 |
| **重定向URL** | 飞书授权回调地址 | 必须与HAP中配置的回调地址一致，确保授权流程正常。 |
| **应用身份权限** | 飞书API接口权限 | 严格按照文档要求勾选所需15项权限，确保HAP能正常调用飞书接口。 |
| **发布版本** | 飞书应用版本管理 | 每次修改配置后需创建新版本并发布，版本号递增，确保更新生效。 |
| **同步人员范围** | 决定哪些飞书用户同步到HAP | 根据实际HAP用户数和部门结构进行精确配置，避免同步无关人员。 |
| **消息打开方式** | HAP消息在飞书中的展示方式 | 可选独立窗口或侧边栏，根据用户习惯和业务需求选择。 |
| **自定义字段“同步”** | 用于控制飞书通讯录同步到HAP的成员 | 将不希望同步的成员此字段设置为“不同步”，实现精细化管理。 |

明道云HAP飞书集成模块主要通过配置方式实现，不涉及直接的二次开发或代码编写。因此，此处不提供开发规范。用户主要关注配置流程和权限管理。

## 6. 最佳实践

*   **提前规划同步范围**：在进行通讯录同步前，务必明确哪些飞书用户需要同步到HAP，并根据实际情况配置同步人员范围，避免不必要的账户创建和数据冗余。
*   **手机号/邮箱一致性检查**：同步通讯录前，提醒HAP现有用户核对HAP账户与飞书账户绑定的手机号和邮箱是否一致，这是关联账户的关键，可有效减少重复账户或关联失败的问题。
*   **利用自定义字段排除**：对于不希望同步到HAP的飞书成员（如实习生、兼职），利用飞书的自定义字段“同步”功能进行排除，实现通讯录的精细化管理。
*   **及时发布应用版本**：飞书应用配置有任何变更后，务必及时创建新版本并发布，并确保管理员审核通过，以保证新配置的生效。
*   **测试验证集成效果**：完成集成配置后，务必进行全面的测试，包括通讯录同步、消息推送、工作台应用访问、审批流程同步及飞书扫码登录等，确保各项功能正常运行。
*   **关注权限最小化原则**：在飞书应用中配置API权限时，遵循最小化原则，只授予HAP集成所需的必要权限，避免过度授权带来的安全风险。
*   **妥善保管App ID和App Secret**：App ID和App Secret是飞书应用的重要凭证，需妥善保管，防止泄露。
*   **理解HAP只能对接一个平台**：如果企业同时使用钉钉和飞书，需要明确HAP只能对接其中一个平台。若需切换，需先关闭已集成的平台。

## 7. 与其他模块的配合

飞书集成模块与HAP的多个核心模块紧密配合，共同提升企业运营效率：

*   **组织管理模块**：飞书的组织架构同步到HAP后，直接更新HAP的组织管理模块，为HAP内的用户、角色、权限管理提供基础数据。
*   **工作流模块**：HAP工作流产生的待办和通知，通过飞书集成模块推送到飞书，实现工作流消息的跨平台触达，并可与飞书审批中心联动，加速审批流程。
*   **应用模块**：HAP中的各种业务应用（如销售管理、项目管理、人事管理等）可以作为飞书工作台的应用，为飞书用户提供统一的应用入口，提升应用的可访问性。
*   **用户角色模块**：飞书同步过来的用户可以在HAP中被分配到不同的用户角色，从而获得相应的操作权限，确保数据安全和业务流程的合规性。

## 8. 实际案例

### 8.1 案例一：销售订单审批流程优化

**业务背景**：某销售公司使用HAP管理销售订单，销售人员在HAP中提交订单后，需要销售经理审批。销售经理日常主要使用飞书进行沟通和管理。

**集成方案**：
1.  配置HAP与飞书集成，启用机器人功能。
2.  将HAP中的销售订单审批工作流配置为在审批节点向销售经理推送飞书消息。
3.  将HAP的销售管理应用添加到飞书工作台。

**效果**：销售人员在HAP提交订单后，销售经理立即在飞书收到审批通知，点击通知即可跳转到HAP的审批页面进行处理。销售经理也可以直接从飞书工作台进入销售管理应用查看订单详情，大大缩短了审批周期，提高了订单处理效率。

### 8.2 案例二：新员工入职流程自动化

**业务背景**：公司新员工入职时，HR需要在飞书和HAP中分别创建员工信息，并进行通讯录同步，流程繁琐且易出错。

**集成方案**：
1.  配置HAP与飞书通讯录同步功能。
2.  在飞书创建新员工信息后，HAP自动同步创建员工账户。
3.  HAP中的入职流程工作流配置相关任务通知推送到飞书。

**效果**：HR只需在飞书录入新员工信息，HAP便自动创建账户并同步通讯录。入职流程中的各项任务（如领取办公用品、参加培训等）通过飞书消息通知到新员工，新员工可以直接在飞书查看任务详情，提升了入职效率和体验。

### 8.3 案例三：考勤打卡与数据统计

**业务背景**：公司使用HAP进行员工考勤管理，员工需要每天登录HAP进行打卡。希望能在飞书上直接完成打卡操作。

**集成方案**：
1.  配置HAP与飞书集成，将HAP的考勤应用集成到飞书工作台。
2.  HAP考勤应用支持飞书扫码登录。

**效果**：员工每天打开飞书，直接在工作台点击考勤应用图标即可进入HAP考勤页面进行打卡，或通过飞书扫码快速登录HAP完成打卡。HR部门可以在HAP中实时查看考勤数据并进行统计分析，无需额外操作，提高了考勤管理的便捷性。
