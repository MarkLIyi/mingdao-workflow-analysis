# 明道云帮助文档研究笔记

## HAP 平台介绍

**HAP (Hyper Application Platform)** - 超级应用平台

### 核心定位
- 零代码构建企业应用平台
- 无需代码开发即可搭建销售、运营、人事、采购等核心业务应用
- 超自动化引擎,自动化复杂和重复的业务流程
- 集成中心与完整API对接能力
- 支持云原生架构,多云部署能力
- 插件架构,建立实施与开发生态

### 八大功能模块

1. **工作表** - 收集和存储业务数据(员工表、客户表、订单表、库存表等)
2. **视图** - 业务数据的不同场景展示页面
3. **角色和权限** - 控制成员对业务数据增删改查的操作权限
4. **工作流** - 实现业务数据的自动化处理和自动流转
5. **对话机器人** - 以多模态对话方式调用数据、流程与外部应用
6. **统计图表** - 统计多种维度的数据帮助分析决策
7. **自定义页面** - 放置快捷按钮、统计图表,配置仪表台、驾驶舱或工作台
8. **外部门户** - 邀请外部人员加入使用,打通内外用户

### 应用场景
- 替代传统定制开发
- 帮助ISV构建垂直行业解决方案
- 替换老旧IT系统
- 企业数据管理中心和流程引擎
- 全民数字化赋能工具

### 部署方式
- SaaS平台开箱即用
- 可部署在任何云计算环境中
- 包含丰富的管理和安全特性,满足大型组织和特定行业的合规需求

## 快速入门核心流程

### 应用搭建前的准备
搭建应用前需要梳理三个核心事项:
1. 要实现的业务流程和自动化流程
2. 流程涉及到的所有业务数据
3. 流程涉及人员/角色及对应的操作权限

### 工作表核心特性
- 类似在线Excel,用于管理业务数据
- 每一行数据(记录)都可以进行讨论协作
- 支持分享到其他平台查看(微信、QQ等)
- 用表单控件快速建立数据结构
- 支持建立不同表之间的关联实现数据互通互联
- 支持Excel、API向工作表写入数据
- 支持免登录公开收集数据

### 工作表可管理的数据类型
只要可以用Excel或纸张管理的数据都可以建立对应的工作表,例如:员工表、档案表、客户表、订单表、产品表、工单表等

### 工作表核心功能
1. 创建/修改工作表
2. 记录操作:添加、修改、删除、打印、分享、讨论
3. 批量操作:批量导出或删除记录、批量修改某个字段的值或清空原值
4. 数据导入导出:Excel表数据导入工作表、导出表数据到Excel
5. 统计数据:表统计

### 工作流核心概念

**定义**: 为业务流程和审批流程提供一个可自动执行的流程模型

**组成**: 一个工作流由触发器和若干个动作节点组成
- 触发器:流程启动的开关
- 动作节点:完成各种任务

**特点**:
1. 完全在互联网上运行,可以接通外部数据源
2. 创建和维护过程友好,不需要掌握艰深的IT知识
3. 强调自动化的触发和执行

**应用场景**:
1. 数据的自动化处理
   - 当数据变更满足条件时,新增或修改其他表的数据
   - 当设定的时间到达时,修改某些记录的数据

2. 业务流程的审批控制
   - 销售订单、项目变更等业务审批
   - 请假、用人、物品领用等事务审批

3. 通知内部和外部人员查看或填写数据
   - 以系统消息的方式通知内部成员数据的变更
   - 将记录内容以邮件、短信的方式发送给外部用户

4. 和外部系统的数据对接
   - 将工作表的数据推送到外部系统
   - 主动或被动的获取外部系统数据,写入或更新到工作表

## 关键设计规范

### 工作表设计规范
1. 使用40种控件来设计工作表字段
2. 支持关联记录控件建立表间关联关系
3. 支持业务规则:根据字段A的值控制其他字段的显示/隐藏/可编辑性
4. 支持自定义动作:添加按钮弹出窗口填写指定字段
5. 支持打印模板:基于Word模板打印记录
6. 支持公开表单:对外发布二维码或链接收集数据

### 视图设计规范
- 根据业务场景和操作人员展示不同的数据
- 支持多种视图类型(列表、日历等)
- 可按处理进度等维度管理数据

### 角色权限规范
- 控制成员对业务数据增删改查的操作权限
- 需要邀请同事注册账号
- 为同事分配角色和权限

### 统计图表规范
- 用于统计多种维度的数据帮助分析决策
- 可配置图表类型和数据源

### 自定义页面规范
- 可放置快捷按钮、统计图表
- 常用于配置仪表台、驾驶舱或工作台
- 支持添加多个统计图

## API 专区核心知识

### API 分类
明道云平台提供2类API:
1. **组织相关接口** - 涵盖人员、部门、汇报关系、日志、待办等数据
2. **应用接口** - 应用创建后自动创建对应的API接口

### 组织相关接口
- 查看路径: 组织管理 > 集成 > 其他 > 开放接口
- 查看组织密钥: AppKey、SecretKey
- 文档地址(SaaS): https://apidoc.mingdao.com
- 私有部署: 调试地址自动显示为私有部署域名

### 应用接口

**文档入口**: 点击应用名称右侧按钮 > API开发文档

**应用授权**: 可创建多个应用授权,每个授权包含AppKey、Sign、接口权限范围

**API版本**:
- **API V2**: 每个工作表自动生成直接可用的接口,请求示例中自动写入appKey、Sign、worksheetId等参数
- **API V3**: 在V2基础上重构,参数命名更友好便于AI识别,新增更多能力
  - 新增/编辑/删除工作表API
  - 工作流API
  - 工作表数据聚合API
  - 公共查询API(查人员、部门)
  - 文档地址: https://apidoc.mingdao.com/application

**字段和视图对照表**:
- 字段对照表: 字段ID、字段名称、类型、控件类型编号、说明
- 视图对照表: 视图ID、视图名称、类型

**接口分享和调试**:
- 在接口文档右上角可进行调试和分享
- 分享时可设置链接有效期和密码访问
- 分享的API接口页面中密钥信息不会显示

**筛选条件生成器**:
- 通过可视化配置生成筛选代码
- 入口: 应用开发文档 > 应用授权 > 工作表 > 目标工作表 > 字段对照表 > 筛选条件生成器

### API调用限流与限制

**限流规则**: 仅按同一IP限制调用频率,不按应用或用户维度限流

| 接口 | 单个IP的QPS(每秒请求数) | 请求体大小限制 |
|------|------------------------|---------------|
| 新建行记录(addRow) | 50 | 16MB |
| 批量新增行记录(addRows) | 50 | 16MB |
| 编辑行记录(editRow) | 50 | 16MB |
| 批量编辑行记录(editRows) | 50 | 16MB |
| 删除行记录(deleteRow) | 50 | 16MB |
| 获取记录数量(getFilterRowsTotalNum) | 50 | / |
| 获取记录列表(getFilterRows) | 50 | / |
| 其他接口 | 不限制 | / |
| 私有部署 | 不限制 | 25MB |

**超限处理**:
- 超过调用频率限制: 返回HTTP状态码503,错误码90001
- 超过请求体大小限制: 返回HTTP状态码413

**应对策略**:
- 加入重试机制
- 在请求中加入时间间隔,如Sleep(200ms)
- 数据拆分: 将大数据集拆分成多个较小部分分别发送
- 优化数据结构: 移除不必要字段,压缩数据结构
