# 明道云代码块节点模块研究报告

## 1. 核心概念和术语定义

*   **代码块节点（Code Block Node）**: 工作流中的一个节点，允许用户输入JavaScript或Python代码，对流程中的记录数据进行个性化处理、计算，并输出新的数据内容供后续节点使用。
*   **输入参数（Input Parameters）**: 传递给代码块节点进行处理的数据。在代码中通过 `input.参数名` (JavaScript) 或 `input["参数名"]` (Python) 访问。
*   **输出参数（Output Parameters）**: 代码块节点处理完成后返回的数据。需要按照特定格式 `output = {参数名1:参数值1,参数名2: 参数值2}` (JavaScript) 或 `output = {'参数名1':canshu1,'参数名2': canshu2}` (Python) 进行定义。
*   **代码片段库（Code Snippet Library）**: 存储常用代码片段的地方，用户可以保存和复用代码，支持组织和个人维度。
*   **AI生成代码（AI Generated Code）**: 通过集成AI能力，根据功能需求描述自动生成符合代码块规范的JavaScript或Python代码。

## 2. 完整的功能列表和使用方法

### 2.1 节点功能

*   在节点内输入JavaScript或Python代码，对流程中的记录数据进行个性化处理计算。
*   将处理后的新数据内容输出，供后续节点使用。

### 2.2 支持的编程语言和运行环境

*   **JavaScript**: Node.js v10.16.3
    *   **已安装模块**: `moment`, `lodash`, `node-fetch`, `form-data`, `xml2js`, `node-rsa`, `uuid`, `pinyin`, `sm-crypto`
*   **Python**: Python v3.7.5
    *   **已安装模块**: `requests`, `xmltodict`, `pycryptodome`, `dingtalk-sdk`, `beautifulsoup4`, `rsa`, `uuid`, `xpinyin`, `gmssl`, `cryptography`, `lxml`, `pandas`, `numpy`, `matplotlib`

### 2.3 数据处理能力

*   **可接收的数据类型**: 数值、文本、时间、单选、成员等；数组类型（接收后会当做字符串处理）。
*   **可输出的数据类型**: 文本、数组。
*   **限制**: 如果是“获取多条数据”节点传给代码块的数据，最多接收处理100行。

### 2.4 API调用

*   支持通过代码块调用API，请求支持WebService SOAP协议。
*   **参数格式**: XML/JSON/Form-Data。
*   **请求方式**: GET/POST/PUT/DELETE。

### 2.5 代码片段库

*   **保存到代码片段库**: 将编写好的代码保存起来，供同事或自己下次直接使用。
*   **从代码片段库选择**: 从片段库中选择需要的代码，直接插入到代码块中（可选择是否清空原有代码）。

### 2.6 AI生成代码

*   根据输入的功能需求描述自动生成JavaScript或Python代码。
*   生成的代码符合代码块规范，可直接使用。
*   可选择在使用时清空现有input参数与代码块内容。

## 3. 配置规范和参数说明

### 3.1 传入参数配置

*   **定义传入参数名称**: 在代码块节点配置界面定义。
*   **设置传入参数值**: 支持多个参数传入，可选择固定值或动态值（从其他节点选择）。
*   **代码中接收参数**: 
    *   JavaScript: `input.参数名`
    *   Python: `input["参数名"]`

### 3.2 输出参数配置

*   **代码中输出格式**: 
    *   JavaScript: `output = {参数名1:参数值1,参数名2: 参数值2}`
    *   Python: `output = {'参数名1':canshu1,'参数名2': canshu2}`
*   **参数命名**: 实际使用时，参数名必须使用英文命名，避免使用中文导致不兼容错误。

### 3.3 代码块测试

*   编写代码后，点击测试按钮进行测试。
*   如果参数值选择动态值，需要填写测试值。
*   只有测试成功并保存节点后，后续节点才能使用代码块的输出参数。

## 4. 最佳实践和设计模式

*   **参数命名规范**: 输出参数名应使用英文命名，便于识别和避免兼容性问题。
*   **代码测试**: 及时测试代码块，确保输出符合预期，以便后续节点正常使用。
*   **代码复用**: 利用代码片段库保存和复用常用代码，提高开发效率。
*   **AI辅助**: 结合AI生成代码功能，快速构建符合规范的代码。
*   **错误处理**: 对于可能导致代码块执行失败的情况（如网络通信异常、服务波动、内存占用过高），可以配置自动重试机制（默认重试1次），或取消重试以立即中止流程。

## 5. 常见问题和注意事项

*   **数据量限制**: “获取多条数据”节点传给代码块的数据，最多接收处理100行。
*   **输出参数中文命名**: 虽然示例中使用中文，但实际使用时必须用英文命名输出参数，否则可能导致不兼容错误。
*   **代码编写能力**: HAP不提供代码编写服务，建议用户学习JavaScript或Python语言。
*   **测试的重要性**: 只有代码块节点测试成功并保存后，其输出参数才能被后续节点使用。
*   **自动重试**: 代码块执行失败时会自动重试1次，可根据需求选择是否取消此功能。

## 6. 与其他模块的关联关系

*   **工作流（Workflow）**: 代码块节点是明道云工作流中的一个重要组成部分，用于增强工作流的自定义处理能力。
*   **获取多条数据节点**: 代码块节点可以接收“获取多条数据”节点输出的数据进行处理，但有100行的限制。
*   **API集成**: 代码块节点可用于调用外部API，实现与第三方系统的数据交互。
*   **后续节点**: 代码块节点的输出数据可以作为后续节点的输入，实现数据的链式处理。
*   **子流程**: 文档中提到“超过100 条建议用子流程去处理”，暗示代码块在处理大量数据时可能与子流程节点存在协作或替代关系。
