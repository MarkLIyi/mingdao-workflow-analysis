# 明道云API集成模块研究报告

## 1. 核心概念和术语定义

### API集成模块
在集成中心的API集成模块，可以统一管理第三方API的鉴权认证配置和API接口的请求配置，经过授权的应用都可以直接调用配置好的API来获取数据或推送数据。

### 我的连接
在这里可以创建第三方API的连接并保存鉴权认证、配置接口请求信息和授权给相关应用。

### API库
*   **公共库：** 由平台预设的API连接模板，可以直接安装使用，但是API连接配置不能更改，只能填充参数。
*   **组织库：** 组织管理员可以将组织内创建的API连接添加到组织库，其他成员可以申请使用。

### 连接参数
配置鉴权时依赖的一些身份参数，如企业微信中获取Token时需要的corpid和corpsecret。这些参数服务商都会分配，用户只需粘贴。连接参数也可以在调用API时作为请求参数使用。

### 鉴权认证方式
*   **API Key或无需鉴权：** 如果不需要鉴权或者在请求URL中进行API Key鉴权，则选择此方式。
*   **Basic Auth 认证：** 每次请求API时都需要提供用户的username和password。
*   **OAuth 2.0 鉴权认证(客户端凭证)：** 通过此方式可以获取到的Access Token，供API请求时使用。支持配置过期时长，到时自动获取新Token。
*   **OAuth 2.0 认证（授权码）：** 基于个人身份进行认证，在「账户」模块可以查看和管理已授权账户。

### 输入参数
用于在工作表或工作流中使用 API 查询时，可以传入动态值。传进来后，可以将此参数值再传给API请求参数。

### 输出参数
在API执行请求后，将获取到的数据格式化输出，以便被工作表字段或被工作流节点引用。

### 代码块
当传入的参数格式或数据不符合API参数要求时，可以通过代码处理后再传递给API参数。也可以将API返回的结果经常代码处理后再输出。支持 Javascript 和 Python 语言。

## 2. 完整的功能列表和使用方法

### 2.1 功能概述
*   统一管理第三方API的鉴权认证配置和API接口的请求配置。
*   授权应用可以直接调用配置好的API来获取数据或推送数据。
*   提供自定义配置接口和预置200多个常用API模板。
*   组织管理员可将自定义连接添加到组织下的API库。

### 2.2 调用方式
以下几处可直接调用配置好的连接认证或API接口:
*   工作表的`[API查询]`字段和`[文本识别]`字段
*   控件事件中`[调用已集成]`动作
*   工作流的`[调用已集成API]`节点
*   工作流的`[API认证与连接]`节点

### 2.3 自定义连接操作流程
1.  **创建自定义连接：** 进入集成中心，切换到`[我的连接]`，点击`[+自定义连接]`-`[创建自定义连接]`。
2.  **连接与认证：**
    *   选择鉴权方式：根据所需API服务的鉴权要求选择，如API Key、Basic Auth、OAuth 2.0等。
    *   配置连接参数：填写服务商提供的身份参数（如corpid和corpsecret）。
    *   配置鉴权信息：根据选择的鉴权方式填写相应的认证信息，如用户名密码、Access Token获取配置、授权回调地址等。
3.  **API接口请求配置：** 连接认证配置完成后，切换到`[API管理]`，点击创建API按钮。
    *   **基本信息：** 设置API的名称、描述说明、图标和颜色。
    *   **输入参数：** 配置API的输入参数，用于传入动态值给API请求参数。支持文本、数值、日期时间、人员、部门、附件、数组、对象数组等类型，可设置必填和默认值。
    *   **代码块：** 可使用Javascript或Python对输入输出参数进行个性化处理，如加密Token、处理手机号、附件URL字符串、JSON数据封装等。
    *   **API请求配置：** 配置发送API请求时需要的URL、Query Param、Header、Body等请求参数。参数值可选择`[连接参数]`、`[输入参数]`、`[鉴权认证]`中的动态值。
    *   **输出参数：** 配置API执行请求后获取到的数据格式化输出，以便被工作表字段或工作流节点引用。通常使用JSON解析。
    *   **发布API：** 配置完成后发布API，才能被应用的工作表或工作流调用。
4.  **授权给应用：** 配置连接的拥有者，并授权给相关应用使用。
5.  **调用集成API：** 在工作表字段、控件事件或工作流节点中调用已配置的API。

### 2.4 API库使用方法
1.  **安装预置API：** 在API库中选择需要的API模板（如天眼查），点击查看接口介绍，然后点击安装。
2.  **填写参数：** 根据界面提示填写必要的参数（如token），这些参数通常需要从服务商处获取。
3.  **开启接口：** 找到需要的接口并开启。
4.  **授权应用：** 只有添加了授权的应用，才能使用此连接服务和API功能。

## 3. 配置规范和参数说明

### 3.1 连接参数
*   **用途：** 配置鉴权时依赖的身份参数，或在调用API时作为请求参数。
*   **特点：** 参数值可选择隐藏显示，隐藏后只能修改不能显示。

### 3.2 鉴权认证参数
*   **Basic Auth：** 用户名和密码可直接写固定值或从连接参数中选择。
*   **OAuth 2.0 (客户端凭证)：**
    *   默认生成`app_id`和`app_secret`两个param参数，可更改或删除。
    *   参数值可从连接参数中选择。
    *   可配置过期时长，到时自动获取新Token。
*   **OAuth 2.0 (授权码)：**
    *   授权回调地址：HAP侧生成，用于接收授权完成后的回调信息。
    *   授权认证地址：OAuth 2.0协议中用于接收用户认证请求的URL。
    *   获取/刷新Access Token节点：根据集成侧的接口文档完成对应鉴权配置。

### 3.3 API接口请求配置参数
*   **API基本信息：** 名称、描述说明、图标和颜色。
*   **输入参数：**
    *   **参数类型：** 文本、数值、日期时间、人员、部门、附件、数组、对象数组。
    *   **附件字段格式：** `["URL1","URL2","URL3"...]`。
    *   **必填：** 必填参数在使用时不能为空，非必填可设置默认值。
    *   **对象数组必填验证规则：** 详细规则如文档所述，涉及数组本身和子元素的必填性。
    *   **别名规则：** 仅支持字母、数字、下划线、短横线（减号），作为发送API请求时数组对象子元素的key。
    *   **默认值：** 参数支持默认值。
*   **API请求参数：** URL、Query Param、Header、Body等。参数值可选择`[连接参数]`、`[输入参数]`、`[鉴权认证]`中的动态值。
*   **输出参数：** 依据测试时返回的响应Body中的数据，使用JSON解析配置。

## 4. 最佳实践和设计模式

*   **复用预置API模板：** 对于常用服务，优先使用平台预置的API模板，减少配置工作量。
*   **自定义连接管理：** 将组织内常用的API连接添加到组织库，方便团队成员共享和使用。
*   **参数隐藏：** 对于敏感参数（如token），勾选隐藏显示，增强安全性。
*   **代码块处理：** 充分利用Javascript和Python代码块对输入输出参数进行灵活处理，以适应不同API的格式要求和业务逻辑。
*   **模块化配置：** 一个连接认证内可添加多个API接口请求，实现API的模块化管理和复用。
*   **授权管理：** 严格控制API连接的拥有者和授权应用，确保数据安全和权限管理。

## 5. 常见问题和注意事项

*   **Token获取与管理：** 对于需要token的API，需前往服务商网站注册账户、申请接口并获取token。OAuth 2.0鉴权支持自动获取和刷新Token。
*   **参数值隐藏：** 参数值隐藏后无法再次显示，只能修改，需谨慎操作。
*   **必填参数：** 必填参数在使用时不能为空，否则可能导致API调用失败。
*   **对象数组验证：** 对象数组的必填验证规则较为复杂，需仔细理解和配置。
*   **代码块语言：** 代码块支持Javascript和Python，选择熟悉或适合的语言进行开发。
*   **API发布：** API配置完成后必须发布才能被应用调用。
*   **授权应用：** 只有经过授权的应用才能使用API连接服务和API功能。

## 6. 与其他模块的关联关系

API集成模块与明道云的其他模块紧密关联，主要体现在以下几个方面：

*   **工作表：** API集成模块配置的API可以通过工作表的`[API查询]`字段和`[文本识别]`字段进行调用，实现数据的查询和处理。
*   **控件事件：** 在控件事件中，可以通过`[调用已集成]`动作来触发API集成模块中配置的API，实现特定事件发生时的自动化操作。
*   **工作流：**
    *   `[调用已集成API]`节点：工作流可以直接使用API集成模块中配置好的API，实现业务流程中的数据交互和自动化。
    *   `[API认证与连接]`节点：工作流可以直接使用API集成模块中配置好的「连接与认证」，实现工作流中对外部系统的认证和连接。
*   **应用：** API集成模块中的连接和API需要授权给特定的应用才能被调用，确保了API使用的安全性和权限控制。
*   **CRM应用：** 文档中提到“进入CRM应用，添加API查询字段和工商信息相关的字段”，表明API集成可以为CRM等业务应用提供外部数据支持。
